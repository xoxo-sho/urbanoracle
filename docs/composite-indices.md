# UrbanOracle 合成指標カタログ

複数のデータソースを掛け合わせて算出する独自指標。生データでは見えない「都市のインサイト」を可視化する。

## 計算ルール

- 各入力値は **min-max 正規化（0-100）** してからスコア化
- 正規化: `score = (value - min) / (max - min) × 100`
- 負の影響を持つ指標は反転: `100 - score`
- 重みは合計1.0。重要な指標ほど高い重みを付与
- 出力: 0-100 のスコア（高いほど良い）

## 不動産・投資系

### 1. 投資ポテンシャル指数

**「今買って5年後に値上がりするエリアはどこか」**

```
投資ポテンシャル =
    地価変動率(+)      × 0.25
  + 人口成長率(+)      × 0.25
  + 交通アクセス(+)    × 0.20
  + 災害リスク(−)      × 0.15
  + 再開発計画有無(+)  × 0.15
```

| 必要データ | ソース | 現状 |
|-----------|--------|------|
| 地価変動率 | REINFOLIB / 国土数値情報 | あり |
| 人口成長率 | e-Stat 国勢調査 | あり |
| 交通アクセス | 駅距離 + 路線数 | 一部あり |
| 災害リスク | ハザードマップ | あり |
| 再開発計画 | 東京都都市整備局 | なし（将来） |

**ターゲット:** 不動産投資家、デベロッパー
**可視化:** コロプレスマップ（赤=高ポテンシャル）+ ランキング表
**即実装:** 再開発を除く4指標で可能

### 2. コスパ居住指数（割安度）

**「住みやすさの割に地価が安いお得なエリアはどこか」**

```
コスパ居住 = 居住快適度 / 地価水準 × 100

居住快適度 =
    交通アクセス     × 0.30
  + 生活利便施設密度  × 0.25
  + 緑地率           × 0.15
  + 安全性           × 0.20
  + 医療アクセス     × 0.10
```

| 必要データ | ソース | 現状 |
|-----------|--------|------|
| 交通アクセス | 駅数 + 路線数 | 一部あり |
| 生活利便施設 | OSM（コンビニ、スーパー） | なし |
| 緑地率 | 東京都環境局 | サンプルあり |
| 安全性 | ハザードマップ | あり |
| 医療アクセス | 国土数値情報 医療施設 | なし |
| 地価水準 | REINFOLIB | あり |

**ターゲット:** 住み替え検討者、初めての一人暮らし
**可視化:** バブルチャート（X=地価, Y=快適度, 色=コスパ）
**即実装:** 交通 + 安全 + 緑地 + 地価 の4要素で簡易版可能

### 3. 賃貸利回り推定指数

**「賃貸経営の利回りが高いエリアはどこか」**

```

推定年間賃料 =
    人口密度(+)    × 0.30
  + 駅近スコア(+)  × 0.30
  + 昼間人口比(+)  × 0.20
  + 築年数(-)(新しいほど高賃料) × 0.20
```

| 必要データ | ソース | 現状 |
|-----------|--------|------|

| 駅距離 | 国土数値情報 | 一部あり |

**可視化:** ヒートマップ + 区別ランキング

### 7. 複合災害リスク指数

**「複数の災害が重なるワーストケースのリスクはどこが高いか」**

```
複合リスク = max(洪水Lv, 地震Lv, 津波Lv, 土砂Lv)
           + (該当する災害種類数 - 1) × 補正係数

# 1種類のリスクだけなら max値のみ
# 2種類以上が重なるとペナルティ加算
```

| 必要データ | ソース | 現状 |
|-----------|--------|------|
| 各種災害リスクレベル | ハザードマップ | あり |

**即実装可能:** 既存データのみで計算可能
**可視化:** 地図上で災害の重なりを色の濃さで表現
