# UrbanOracle データ拡充戦略

## 1. 現状の課題

### 粒度が粗い

現在のデータはすべて**区レベル**（23区）。東京23区の面積は最小10km²（千代田区）から最大61km²（大田区）で、区内の差異が一切見えない。例えば渋谷区内でも駅前の商業地域と住宅街では地価が10倍以上異なるが、現状では平均値しか見えない。

| 粒度 | 現状 | 目標 |
|------|------|------|
| 区（23区分） | 現在 | — |
| 1kmメッシュ（約900区分） | 未対応 | Phase 1 |
| 街区・丁目（数千区分） | 未対応 | Phase 2 |
| 建物レベル（数十万区分） | 未対応 | Phase 3 |

### データの種類が少ない

| 分野 | 現状 | 都市分析に必要 |
|------|------|-------------|
| 地価 | 区別平均のみ | 地点別公示地価 + 取引価格 + 時系列 |
| 人口 | 区別人口・年齢構成 | メッシュ別人口 + 世帯構成 + 昼夜間人口比 |
| 災害 | 区別リスクレベル + ハザードタイル | 建物別浸水深 + 地盤データ + 避難所 |
| 交通 | 10駅のみ | 全駅 + バス + 徒歩圏分析 + 通勤流動 |
| 用途地域 | 区別面積割合 | ポリゴン別用途地域 |
| **経済** | なし | 事業所数・従業者数・産業構成 |
| **不動産市場** | なし | 賃料相場・空室率・築年数分布 |
| **環境** | なし | 緑地率・大気質・騒音・日照 |
| **インフラ** | なし | 上下水道・ガス・通信・病院・学校 |
| **将来予測** | なし | 人口予測・開発計画・再開発情報 |

### 時系列がない

現在のデータは単一時点のスナップショットのみ。「この区は成長しているのか衰退しているのか」という最も基本的な問いに時系列データなしでは答えられない。

## 2. 追加すべきデータカテゴリ

### 2.1 経済・産業データ

| データ | 内容 | インサイト |
|--------|------|----------|
| 事業所数・従業者数 | 業種別、メッシュ別 | 商業集積度、オフィス密度 |
| 昼夜間人口比 | 昼間人口 / 夜間人口 | 通勤流入の大きさ（千代田区は17倍） |
| 小売売上高 | 業種別年間売上 | 商圏の強さ |
| 開業率・廃業率 | 区別年次 | 経済の活力度 |

**データ元:** e-Stat 経済センサス（statsDataId: 複数）

```
GET https://api.e-stat.go.jp/rest/3.0/app/json/getStatsData
  ?appId={ESTAT_API_KEY}
  &statsDataId=0003459856  # 経済センサス 事業所数
  &cdArea=13101,13102,...
```

### 2.2 不動産市場データ

| データ | 内容 | インサイト |
|--------|------|----------|
| 不動産取引価格 | 個別取引（REINFOLIB 既存） | 実勢価格の分布 |
| 賃料相場 | 住宅・オフィス別 | 投資利回り計算 |
| 新築着工数 | 構造別・用途別 | 開発トレンド |
| 空き家率 | 区別 | 住宅ストック状況 |

**データ元:**
- REINFOLIB API（既存接続を拡張、全区・複数年分）
- e-Stat 住宅・土地統計調査
- 建築着工統計（国土交通省）

### 2.3 環境データ

| データ | 内容 | インサイト |
|--------|------|----------|
| 緑被率 | 区別・メッシュ別 | 住環境の質 |
| 大気質（PM2.5, NO2） | 測定局データ | 健康リスク |
| 騒音レベル | 幹線道路沿い | 住環境の質 |
| ヒートアイランド | 地表面温度 | 夏季の過ごしやすさ |

**データ元:**
- 環境省 大気汚染物質広域監視システム（そらまめ君）: https://soramame.env.go.jp/
- 東京都環境局 緑被率調査
- NASA Landsat / Sentinel-2 衛星データ（地表温度）

### 2.4 インフラ・生活利便性

| データ | 内容 | インサイト |
|--------|------|----------|
| 医療施設 | 病院・診療所の位置 | 医療アクセス |
| 教育施設 | 学校・保育園の位置 | 子育て環境 |
| 商業施設 | コンビニ・スーパーの密度 | 生活利便性 |
| 公園・公共施設 | 位置と面積 | レクリエーション |
| 避難所 | 位置と収容人数 | 防災対策 |

**データ元:**
- 国土数値情報 医療施設: https://nlftp.mlit.go.jp/ksj/gml/datalist/KsjTmplt-P04-v3_0.html
- 国土数値情報 学校: https://nlftp.mlit.go.jp/ksj/gml/datalist/KsjTmplt-P29-v2_0.html
- OpenStreetMap Overpass API（コンビニ、スーパー、公園）

### 2.5 将来予測・開発計画

| データ | 内容 | インサイト |
|--------|------|----------|
| 人口推計 | 2050年までの区別推計 | 将来の需要予測 |
| 再開発計画 | 大規模開発の位置と時期 | 地価上昇ポテンシャル |
| 新線計画 | 鉄道延伸・新駅 | 交通利便性の変化 |

**データ元:**
- 国立社会保障・人口問題研究所: https://www.ipss.go.jp/
- 東京都都市整備局 都市計画情報
- 各区の都市計画マスタープラン

## 3. 粒度改善ロードマップ

### 3.1 Phase 1: 1kmメッシュ（3次メッシュ）

国土数値情報の1kmメッシュ人口データを導入。

```bash
# ダウンロード
curl -o mesh-pop.zip \
  "https://nlftp.mlit.go.jp/ksj/gml/data/m1000/m1000-20/m1000-20_GML.zip"
```

| 項目 | 値 |
|------|-----|
| メッシュ数 | 東京23区で約900メッシュ |
| データサイズ | 数MB（GeoJSON） |
| 含むデータ | 総人口、男女別、年齢5歳階級別 |
| 表示方法 | MapLibre fill レイヤー（色分け） |
| 実装難度 | 低（既存コロプレスの拡張） |

**新しい型定義:**

```typescript
interface MeshPopulation {
  meshCode: string;
  lat: number;
  lng: number;
  population: number;
  malePopulation: number;
  femalePopulation: number;
  ageGroups: Record<string, number>; // 5歳階級別
}
```

### 3.2 Phase 2: 街区・丁目レベル

小地域統計（国勢調査の町丁字等別集計）を導入。

```
GET https://api.e-stat.go.jp/rest/3.0/app/json/getStatsData
  ?appId={KEY}
  &statsDataId=0003445078
  &cdArea=13101001,13101002,...  # 町丁字等コード
```

| 項目 | 値 |
|------|-----|
| 区分数 | 東京23区で約5,000町丁字 |
| データ | 人口・世帯数・住宅種類 |
| 表示 | ポリゴン塗り分け（e-Stat 境界データ） |
| 実装難度 | 中（境界GeoJSONの取得が必要） |

### 3.3 Phase 3: 建物レベル

国土数値情報の建物データ（3D都市モデル PLATEAU）を導入。

```bash
# PLATEAU (3D都市モデル) CityGML
# https://www.geospatial.jp/ckan/dataset/plateau-tokyo23ku-2023
```

| 項目 | 値 |
|------|-----|
| 建物数 | 東京23区で約80万棟 |
| データ | 建物高さ、構造、築年、用途、階数 |
| サイズ | 数百MB → PMTiles で数十MB |
| 表示 | MapLibre 3D extrusion（高さ表現） |
| 実装難度 | 高（PMTiles必須、パフォーマンス最適化） |

## 5. クロス分析の機会

### 高価値なクロス分析

| 分析 | 掛け合わせるデータ | ユーザーの問い |
|------|-----------------|-------------|
| **投資適性スコア** | 地価変動率 × 人口成長率 × 交通アクセス × 災害リスク | 「今買うべきエリアはどこか」 |
| **子育て環境スコア** | 学校密度 × 公園面積 × 犯罪率 × 医療施設 × 年少人口比 | 「子育てに適したエリアは」 |
| **ビジネス立地スコア** | 昼間人口 × 事業所密度 × 賃料 × 交通 × 用途地域 | 「オフィスを出すならどこか」 |
| **防災レジリエンス** | 浸水リスク × 建物耐震性 × 避難所距離 × 人口密度 | 「災害時に最も危険なエリアは」 |
| **高齢化対応度** | 高齢化率 × 医療施設 × バリアフリー × 商業施設 | 「高齢者が暮らしやすいエリアは」 |

### 実装方法

各スコアは正規化（0-100）した指標の重み付き合計で算出。WardProfile の拡張版として実装し、RadarChart で可視化。

```typescript
interface AreaScore {
  meshCode: string;
  investmentScore: number;    // 投資適性
  familyScore: number;        // 子育て環境
  businessScore: number;      // ビジネス立地
  resilienceScore: number;    // 防災レジリエンス
  seniorScore: number;        // 高齢化対応度
}
```

## 7. 技術的な実装方針

### データ量と配信方式の選択基準

| データ量 | 配信方式 | 例 |
|---------|---------|-----|
| < 1MB | JSON/GeoJSON 直接 | 区別統計、駅リスト |
| 1-10MB | GeoJSON + gzip | メッシュ人口、地価全地点 |
| 10-50MB | PMTiles（静的） | 用途地域ポリゴン |
| 50MB+ | PostGIS + タイルサーバー | 建物データ、PLATEAU |

### キャッシュ戦略

| データ | 更新頻度 | キャッシュ |
|--------|---------|----------|
| 国勢調査 | 5年 | 30日 |
| 地価公示 | 年1回 | 30日 |
| 経済センサス | 年1回 | 30日 |
| ハザードマップ | 不定期 | 7日 |
| リアルタイム交通 | 分単位 | 60秒 |

### 型定義の拡張方針

```typescript
// メッシュレベルのデータ
interface MeshData {
  meshCode: string;
  geometry: GeoJSON.Polygon;
  population: number;
  landUse: string;
  // ... 各種データを統合
}

// POIデータ
interface PointOfInterest {
  id: string;
  name: string;
  category: "hospital" | "school" | "park" | "convenience" | "shelter";
  lat: number;
  lng: number;
  properties: Record<string, string | number>;
}

// クロス分析スコア
interface AreaScore {
  areaId: string; // mesh or ward
  scores: Record<string, number>; // 各スコア 0-100
  rank: Record<string, number>; // 順位
}
```
